- Input validation on all API routes

### Installation

1. Clone the repository
```bash
git clone https://github.com/darshan-gowdaa/petrol-bunk-management-system.git
cd petrol-bunk-management-system
```

2. Install all dependencies
```bash
# Install root + frontend + backend dependencies (all at once)
npm install && npm install --prefix backend && npm install --prefix frontend
```

3. Set up environment variables

Create a `.env` file in the **root** directory:
```env
# MongoDB
MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/petrol-bunk?retryWrites=true&w=majority

# Auth
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=<generate with generateHash.js>
JWT_SECRET=<strong random secret>

# Server
NODE_ENV=development

# Frontend (Vite)
VITE_API_URL=/api
```

> To generate a bcrypt password hash, run:
> ```bash
> node backend/generateHash.js
> ```

4. Start the development servers (frontend + backend concurrently)
```bash
npm start
```

- Frontend: `http://localhost:5173`
- Backend API: `http://localhost:5000`

## вҳҒпёҸ Deployment (Vercel)

This project is pre-configured for Vercel via `vercel.json`.

- Frontend is built as a static site (`@vercel/static-build`)
- Backend runs as a serverless function via `api/index.js` (`@vercel/node`)
- All `/api/*` requests are routed to the serverless backend
- SPA fallback routes all non-asset paths to `frontend/index.html`

**Required Vercel environment variables:**
```env
MONGODB_URI=
ADMIN_USERNAME=
ADMIN_PASSWORD_HASH=
JWT_SECRET=
NODE_ENV=production
VITE_API_URL=/api
```
